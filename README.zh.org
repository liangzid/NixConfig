* 什么是 NixOS？

NixOS 是基于 =Nix= 构建的特殊 Linux 发行版。

Nix 是什么？

Nix 既是：
+ 一门函数式编程语言，也是解决配置问题的 DSL
+ 一个包管理工具
+ 一个操作系统

Nix/NixOS 的核心理念：
+ 系统中的一切都可以被描述，借助函数式定义的特性，系统可以被轻松复现
+ 类似于 =git=，Nix/NixOS 用更高效的方式管理不同版本的包及其依赖——同一个程序的两个版本可以共存
+ 不同于快照方案，NixOS 允许引导任意版本的构建，因此再也不用担心系统崩溃
+ 我使用它的原因：希望配置是 =可积累的=；享受这种方式，就像我喜欢用 Emacs 一样

** 硬件配置（至关重要）

=hosts/nixos/hardware-configuration.nix= 是*机器相关的*。它包含磁盘布局
（分区UUID）、内核模块、CPU 类型等。在另一台机器上套用此文件会导致：

- 无法启动（磁盘 UUID 错误）
- 缺少硬件所需的内核模块
- 可能使用错误的 CPU 微码

*每台新机器都必须重新生成*一份：

#+begin_src bash
  # 在目标机器上（或从安装 ISO 启动，/mnt 已挂载时）：
  nixos-generate-config --root /
  # 将新生成的硬件配置替换仓库中的：
  cp /etc/nixos/hardware-configuration.nix ~/code/NixConfig/hosts/nixos/hardware-configuration.nix
#+end_src

** 引导方式：UEFI vs BIOS

此配置使用 =systemd-boot= 和 =boot.loader.efi.canTouchEfiVariables = true=，
仅适用于 UEFI 系统。如果你的机器是传统 BIOS 引导：

- =boot.loader.systemd-boot= 无效（或静默失败）
- =boot.loader.efi.canTouchEfiVariables= 需要 =/sys/firmware/efi= 目录（BIOS 下不存在）
- 安装后无法启动

如果你使用 BIOS（或无论如何都想用 GRUB），将 [[./hosts/nixos/configuration.nix][configuration.nix]] 第15-17行替换为：

#+begin_src nix
  # UEFI（当前配置 — 删掉）：
  boot.loader.systemd-boot.enable = true;
  boot.loader.efi.canTouchEfiVariables = true;

  # BIOS / GRUB（替换为）：
  boot.loader.grub.enable = true;
  boot.loader.grub.device = "/dev/sda";        # 或 /dev/nvme0n1 — 不带分区号
  boot.loader.grub.useOSProber = true;         # 可选：检测其他操作系统
#+end_src

在运行中的系统上确认引导方式：

#+begin_src bash
  ls /sys/firmware/efi   # 存在 → UEFI；失败 → BIOS
#+end_src

** NVIDIA 显卡

如果你没有 NVIDIA 显卡，请从 [[./hosts/nixos/configuration.nix][configuration.nix]] 第7行*删除* =nvidia.nix= 导入：

#+begin_src nix
  imports = [
    ./hardware-configuration.nix
    ../../modules/system/hyprland.nix
    ../../modules/system/nvidia.nix     # ← 删除或注释此行
    ...
  ];
#+end_src

同时从 [[./hosts/nixos/home.nix][home.nix]] 第113-119行删除 NVIDIA 相关的环境变量
（=LIBVA_DRIVER_NAME=、=GBM_BACKEND=、=__GLX_VENDOR_LIBRARY_NAME=、=WLR_NO_HARDWARE_CURSORS=）。

** CPU 微码

硬件配置可能会设置 =hardware.cpu.amd.updateMicrocode= 或
=hardware.cpu.intel.updateMicrocode=。这些由 =nixos-generate-config=
自动设置。除非你清楚自己在做什么，否则不要手动覆盖。

* 快速开始

** 方法 A：先用图形化安装器安装（更简单）

如果你不想手动敲命令分区，可以先让 NixOS 图形化安装器（Calamares）装好一个基础系统，再在此基础上应用此 flake：

1. 启动 NixOS ISO。在引导菜单选择任意桌面环境（如 Xfce、KDE）——
   这仅影响*安装时的操作环境*，不影响最终系统。

2. 运行图形化安装器。让它自动完成分区、用户创建、时区设置等。
   此时也可以选任意桌面环境 — flake 应用后会将其替换。

3. 重启进入新安装的系统。

4. 克隆本仓库并应用 flake：

   #+begin_src bash
     git clone https://github.com/liangzid/nix-config.git ~/code/NixConfig
     # 为当前机器生成硬件配置：
     sudo nixos-generate-config --root /
     cp /etc/nixos/hardware-configuration.nix ~/code/NixConfig/hosts/nixos/hardware-configuration.nix
     # 检查并调整配置（参见上面的机器相关配置清单）
     git -C ~/code/NixConfig add -A
     sudo nixos-rebuild switch --flake ~/code/NixConfig#nixos
   #+end_src

5. 重启。系统现在完全由此 flake 管理 — TTY 登录，输入 =Hyprland= 或 =gui= 即可启动桌面环境。

原理：安装器在 =/etc/nixos/configuration.nix= 中写入了传统配置，但
=nixos-rebuild switch --flake= 会声明式地替换一切 — 引导程序、包、服务、
桌面环境 — 全部由 flake 决定。图形化安装器所做的磁盘布局则被
=hardware-configuration.nix= 捕获并保留。

** 方法 B：从 NixOS ISO 手动安装（完全控制）

此路径完全在终端中操作 — 分区、格式化、挂载、安装 — 只使用此 flake 和
=bootstrap.sh=。

1. 下载 [[https://nixos.org/download][NixOS ISO]]，烧录到 U 盘并启动。

2. 确认引导方式。分区方案取决于此：

   #+begin_src bash
     ls /sys/firmware/efi   # 存在 → UEFI；"No such file" → BIOS/Legacy
   #+end_src

   这反映了安装 ISO 自身的引导方式，也决定了最终系统的引导方式。
   UEFI 模式需创建 EFI 系统分区（ESP），BIOS 模式则使用 MBR + GRUB。

3. 分区。先用 =lsblk= 确认磁盘名，再用 =parted=、=fdisk= 或 =cfdisk=。
   UEFI 示例：

   #+begin_src bash
     parted /dev/nvme0n1 -- mklabel gpt
     parted /dev/nvme0n1 -- mkpart ESP fat32 1MiB 1024MiB
     parted /dev/nvme0n1 -- set 1 esp on
     parted /dev/nvme0n1 -- mkpart primary ext4 1024MiB 100%
   #+end_src

   BIOS 模式使用 MBR 分区表，不需要 ESP 分区。

4. 格式化并挂载：

   #+begin_src bash
     # UEFI
     mkfs.fat -F 32 /dev/nvme0n1p1
     mkfs.ext4 /dev/nvme0n1p2
     mount /dev/nvme0n1p2 /mnt
     mkdir -p /mnt/boot
     mount /dev/nvme0n1p1 /mnt/boot

     # BIOS（单个 ext4 分区 /dev/sda1）
     mkfs.ext4 /dev/sda1
     mount /dev/sda1 /mnt
   #+end_src

5. 编辑 =bootstrap.sh=，设置配置目录：

   #+begin_src bash
     REPO_URL="https://github.com/liangzid/nix-config.git"
     CONFIG_DIR="/mnt/home/zi/code/NixConfig"
   #+end_src

6. 运行安装脚本：

   #+begin_src bash
     chmod +x bootstrap.sh
     ./bootstrap.sh install
   #+end_src

   脚本将依次：生成硬件配置 → 克隆仓库 → 替换为刚生成的硬件配置 →
   执行 =nixos-install --flake=。安装过程中会提示为 =zi= 用户设置密码。

7. 安装完成后重启。

* 常见错误与排查

** "error: file 'hosts/nixos/hardware-configuration.nix' not found in the Nix store"

flake 要求所有引用的文件被 Git 追踪。你可能新增或替换了
=hardware-configuration.nix= 但没有 git add。

#+begin_src bash
  git -C ~/code/NixConfig add -A
  git -C ~/code/NixConfig commit -m "update hardware config"
#+end_src

** "mount: /boot: unknown filesystem type 'vfat'" 或启动卡住

你用了 UEFI 配置但你的机器是 BIOS/Legacy 引导。将引导程序切换为 GRUB
（见上文 Bootloader 章节）。如果已经安装了，重新启动 NixOS ISO，将
根分区挂载到 /mnt，修改 =/mnt/etc/nixos/hosts/nixos/configuration.nix=
使用 GRUB，然后重新运行 =nixos-install --flake=。

** "error: attribute 'nvidia' missing" / NVIDIA 构建失败

如果你没有 NVIDIA 显卡：
1. 从 =hosts/nixos/configuration.nix= 的 imports 中删除 =../../modules/system/nvidia.nix=。
2. 从 =hosts/nixos/home.nix= 中删除 NVIDIA 相关的环境变量。

如果你有 NVIDIA 但构建失败，尝试切换驱动版本：

#+begin_src nix
  # 在 modules/system/nvidia.nix 中：
  hardware.nvidia.package = config.boot.kernelPackages.nvidiaPackages.stable;
  # 尝试改为：
  # hardware.nvidia.package = config.boot.kernelPackages.nvidiaPackages.beta;
#+end_src

** 引导程序未安装 / 重启后 "no bootable device"

通常这意味着：
- UEFI：=nixos-install= 时 ESP 没有挂载到 =/boot=，或者 ESP 分区类型不对
  （gdisk 中应为 =ef00=，parted 中应为 "EFI System"）。
- BIOS：GRUB 没有安装到 MBR。确保 =boot.loader.grub.device=
  指向整块磁盘（如 =/dev/sda=），而不是分区（=/dev/sda1=）。

修复方法：启动 ISO，和之前一样挂载所有分区，重新运行 =nixos-install --flake=。

** Home-manager 报 "user zi not found" 错误

=home-manager= 配置引用的是用户 =zi=，但你改了用户名。请在
=configuration.nix=（=users.users.*=）和 =home.nix=
（=home.username=、=home.homeDirectory=）*两处*同步修改。

** "ServerAliveCountMax 9999999 is not a valid integer"

较旧的 SSH 客户端不接受过大的 =ServerAliveCountMax= 值。
在 [[./hosts/nixos/home.nix][home.nix]] 中改为 =ServerAliveCountMax 999=。

** Clash / 代理重建后无法使用

=clash-verge-rev= 需要订阅链接才能工作。它已被安装但不会预先配置你的订阅。
打开 Clash → Profiles → 添加你的订阅。然后设置 Merge 指向
=~/.config/clash-verge-rev/= 中对应的 yaml 文件。

** Hyprland 无法启动（只有 TTY） / "start-hyprland" 找不到

Hyprland 已安装，但默认会话是 TTY。登录后运行：

#+begin_src bash
  Hyprland
#+end_src

或使用 =gui= 别名（=start-hyprland=），前提是系统已用此配置构建过
（别名在 =configuration.nix= 中定义）。

** Wofi / 启动器不显示输入法候选词

确保 =fcitx5= 已在运行（在 [[./dotfiles/hypr/hyprland.conf][hyprland.conf]] 的 Hyprland 自启动中加入
=fcitx5 -d=）。环境变量 =GTK_IM_MODULE=fcitx= 等已在 =home.nix= 中配置，
但部分应用需要 =fcitx5= 先启动。

** WeChat / QQ 字体乱码或缺失

[[./modules/system/fonts.nix][fonts.nix]] 模块安装了中文字体。此外，WPS 字体由 =home.nix=
复制到 =~/.local/share/fonts/wps-fonts=。如果字体仍未生效，运行：

#+begin_src bash
  fc-cache -fv
#+end_src

* 注意事项

** Super+L 与 logind

Systemd-logind 默认拦截 =Super+L=。此配置通过
=services.logind.settings= 忽略这些硬件按键，交由 Hyprland 处理。
参见 [[./hosts/nixos/configuration.nix][configuration.nix]]。

** Swaylock PAM

=Swaylock= 配置了 PAM 服务（=security.pam.services.swaylock=），
可以在不需要密码策略覆盖的情况下解锁屏幕。

** Steam

Steam 在系统级别启用：=programs.steam.enable = true=。
这会自动安装所有必需的 32 位库和 Steam 本体 — 无需将 =steam= 加入包列表。

** Ghostty Terminfo

Ghostty 的 terminfo 通过 =environment.sessionVariables.TERMINFO_DIRS=
提供。此外，Ghostty 配置中 =term = xterm-256color= 保证 SSH 到远程服务器
（没有 Ghostty terminfo 条目）时的完全兼容性。

** pi-coding-agent (PI_OFFLINE)

Pi 通过 nixpkgs 作为系统软件包安装。环境变量 =PI_OFFLINE=1= 在
[[./hosts/nixos/home.nix][home.nix]] 中设置，防止 agent 在启动时检查更新 —
Nix 管理的包位于只读的 =/nix/store= 中，无法自行更新。
通过正常的 flake 周期更新 pi：=upgrade=。

** 非自由软件包

=nixpkgs.config.allowUnfreePredicate= 仅白名单实际使用的非自由软件包
（NVIDIA 驱动、Steam、Discord、VS Code、WeChat、QQ 等）。

** Git 要求

flake 要求所有引用的文件被 Git 追踪。重建前请运行：
#+begin_src bash
  git -C ~/code/NixConfig add -A
#+end_src
